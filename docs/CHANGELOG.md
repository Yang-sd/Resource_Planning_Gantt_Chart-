# 分支：feat/single-container-image

- 新增 `Dockerfile.all-in-one`，支持把前端、Flask API 和 MySQL 打包到一个镜像里。
- 新增单容器启动、停止和冒烟测试脚本：`docker-single-up.sh`、`docker-single-down.sh`、`docker-single-smoke.sh`。
- 新增 GitHub Actions 发布流程，推送后可发布到 `ghcr.io/yang-sd/resource-planning-gantt-chart`。
- 部署文档补充“拉镜像直接运行”和“源码构建单容器”两种极速部署路径。

# 版本迭代记录

## v1.9.0

- 组织管理的团队项目和成员负责项目卡片新增“转交”按钮。
- 项目转交弹窗支持选择新负责人，并自动同步项目所属团队。
- 转交保存后写入操作记录，并将组织管理详情切换到新负责人。
- 补齐项目 README、部署文档、架构文档、测试文档和脚本注释。

## v1.8.0

- 新增个人资料头像上传，支持 PNG/JPG/WebP/GIF，最大 10MB。
- 上传头像存储到 MySQL，并通过独立头像接口加载，避免首屏 JSON 过大。
- 修改密码不再要求填写当前密码，只需要新密码和确认新密码。
- 账号卡片和成员头像支持图片头像与文字头像 fallback。

## v1.7.1

- 新建成员时自动创建登录账号，初始密码统一为 `123456`。
- 当前账号卡片支持打开个人资料弹窗。
- 个人资料保存会同步成员档案、团队负责人名称和操作记录。

## v1.7.0

- 新增账号登录与角色权限体系。
- 管理员拥有全部功能，团队负责人可以进入组织管理，普通成员不显示组织管理菜单。
- 后端对组织管理写入接口增加权限校验。

## v1.6.x

- 组织管理升级为独立页面。
- 团队视图和成员视图采用左侧目录加右侧详情结构。
- 总览和项目清单继续压缩信息密度，减少无效留白。
- 项目执行周期列、优先级视觉、负责人识别和版本记录持续优化。

## v1.5.x

- 资源排期改为固定工作台布局。
- 成员行支持上下拖拽排序。
- 资源排期纵向滚动改为局部滚动，避免整页大滚动。

## v1.4.x

- 组织管理从弹窗迁移为独立页面。
- 支持团队、成员新增、编辑、删除和安全校验。
- 新增成员入口移动到团队详情操作区。

## v1.3.x

- 默认标记法定节假日和调休工作日。
- 总览列表支持批量选择和批量删除。
- 项目清单新增项目执行周期列。

## v1.2.x

- 项目条支持停留提示、整体拖动、首尾拉伸和上下移动负责人。
- 资源排期支持按天丝滑横向浏览。
- 今天日期加入淡色高亮。

## v1.0.x
