# 产品、UI/UX 与开发说明

## 产品定位

本项目是一个人力资源项目排期工作台，核心目标是让团队负责人和交付管理者在一个页面里快速回答：

- 当前有哪些项目在执行？
- 每个项目由谁负责？
- 这个人当前负载是否过高？
- 项目周期是否需要平移、拉长或转交？
- 团队和成员关系是否清晰？

它不是完整的企业项目管理平台，也不是 Jira/飞书项目的替代品。当前阶段更聚焦于“轻量、直观、可部署、可维护”的项目排期和人力视图。

## 目标用户

- 小团队负责人
- 产品经理、项目经理、PMO
- 交付负责人
- 设计、研发、运营协作团队

## 核心场景

- 每周排期会前查看近两周或一个月的人力安排。
- 某成员负载过高时，把项目转交给其他成员。
- 新成员加入后，把他录入组织管理并自动生成账号。
- 项目延期时，直接在甘特图上拖拽或拉伸时间。
- 负责人需要追踪谁做了新增、删除、转交、导出等动作。

## UI/UX 原则

- 信息密度要高，但不能变成表格墙。
- 关键操作要就地完成，减少弹窗和页面跳转。
- 左侧导航保持紧凑，把工作区空间留给排期。
- 时间线以天为最小浏览单位，左右移动要平滑。
- 组织管理采用主从结构：左侧目录，右侧详情。
- 项目转交、删除、保存等高影响动作需要明确反馈。
- 暗夜模式要保持可读性，不只是简单反色。

## 页面说明

### 总览

用于快速筛选项目，查看负责人、状态、优先级、执行周期和里程碑。

### 资源排期

核心甘特图页面。支持：

- 双周和一个月时间范围。
- 今日淡色提示。
- 法定节假日和调休默认标记。
- 项目拖动、拉伸、上下移动负责人。
- 鼠标和触控板浏览。

### 组织管理

用于维护团队和成员。支持：

- 团队视图。
- 成员视图。
- 新增、编辑、删除团队。
- 新增、编辑、删除成员。
- 新增成员自动创建登录账号。
- 成员详情中直接转交项目。

### 记录中心

用于查看：

- 版本更新记录。
- 操作记录。

记录中心帮助团队知道系统变更和业务操作发生过什么。

## 开发原则

- 前后端分离，前端只通过 HTTP API 获取和写入数据。
- 后端统一使用 camelCase JSON，降低前端转换成本。
- 数据库存储真实日期，不再使用固定基准日加偏移量作为持久化模型。
- 关键写操作都生成操作记录。
- Docker 部署必须能在没有 Compose 的机器上运行。
- 测试尽量贴近生产环境，后端使用真实 MySQL。

## 后续规划建议

- 大数据量下按日期范围懒加载项目。
- 增加成员负载阈值和风险提醒。
- 增加项目评论、附件和变更历史。
- 增加更细的角色权限，例如只能编辑自己负责项目。
- 增加团队维度容量统计和跨月视图。
- 生产环境接入统一登录或企业身份系统。
